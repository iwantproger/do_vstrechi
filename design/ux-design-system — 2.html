01 — Product UX Overview
Концепция и дизайн-направление
Schedule Booking Mini App — лаконичный, скоростной инструмент синхронизации встреч внутри Telegram. Дизайн ориентирован на нулевой порог входа: клиент записывается за 3 касания, специалист тратит 2 минуты на настройку расписания.
Aesthetic Direction
Dark Minimal
+ Electric Accent
Тёмный фон со сдержанными поверхностями. Синие акценты в стиле Telegram. Чистая типографика. Нет шума, нет лишних элементов.
UX Philosophy
Tap-First,
Zero Friction
Максимум — 3 тапа до подтверждения встречи. Автозаполнение из Telegram. Умные дефолты. Одно действие на экран.
Key Constraint
360px
Mobile-Only
Telegram WebView. Только вертикальная раскладка. Нет URL-бара. Thumb-friendly зоны. Загрузка <2 сек на 3G.

05 — Interaction Patterns
Принципы взаимодействия
UX-правила и паттерны поведения, которые делают приложение быстрым и понятным.
8 основных принципов
  • 01
    Одно действие на экран. Каждый экран ведёт к одному чёткому решению. Экран 3 — выбор даты и слота. Экран 4 — подтверждение данных. Не смешивать.
  • 02
    Автозаполнение везде, где возможно. Имя и контакт из Telegram InitData. Таймзона по IP. Первая опция платформы выбрана по умолчанию. Пользователь только проверяет, не вводит.
  • 03
    Видимый статус системы. Spinner при каждой загрузке. Кнопка "Забронировать" меняет текст на "Отправляем..." при submit. Слот выделяется синим сразу при выборе — без задержки.
  • 04
    Thumb-friendly зоны. Все интерактивные элементы минимум 44px высотой. Слоты 44px. Кнопки 56px. Ничего в верхних 40px экрана (вне досягаемости большого пальца).
  • 05
    Деструктивные действия требуют подтверждения. Отмена встречи → modal с текстом последствий. Нет случайных отмен. Modal закрывается только кнопкой "Продолжить" или "Отмена".
  • 06
    Disabled кнопки объясняют себя. "Продолжить" disabled и полупрозрачна, пока слот не выбран. "Забронировать" disabled, пока форма невалидна. Tooltip не нужен — состояние экрана объясняет блокировку.
  • 07
    Real-time валидация при blur. Ошибка под полем появляется только после того, как пользователь кликнул вне поля. Не при вводе — это раздражает. Только при blur или submit.
  • 08
    Haptic feedback через Telegram SDK. tg.HapticFeedback.impactOccurred('medium') при выборе слота. tg.HapticFeedback.notificationOccurred('success') при успешном бронировании. Тактильная обратная связь без визуального шума.

06 — Errors & Edge Cases
Ошибки и граничные состояния
Полный реестр всех ошибочных состояний, пустых экранов и граничных случаев с UX-решением для каждого.
HTTP ошибки
409
Слот уже забронирован
Кто-то занял слот между выбором и отправкой формы. Показать: "❌ Этот слот уже занят!" + кнопка "← Выбрать другой слот" → возврат на SCREEN_003.
422
Ошибка валидации данных
Показать inline ошибки под каждым полем. Красная граница инпута. "❌ Минимум 2 символа" / "❌ Некорректный email". Без алерта — всё inline.
404
Расписание не найдено
schedule_id не существует или деактивирован. Полноэкранное сообщение: "😔 Расписание не найдено. Возможно, специалист его удалил." + кнопка "← Вернуться".
401
Сессия истекла
Telegram InitData протухла. "❌ Сессия истекла. Пожалуйста, откройте приложение заново через Telegram." + кнопка "🔄 Перезагрузить".
500
Ошибка сервера
"⚠️ Что-то пошло не так. Попробуйте ещё раз через пару минут." + [🔄 Повторить] [← Назад]. Логировать в консоль для отладки.
Сетевые и граничные случаи
КейсUX-решение
Нет интернетаToast снизу "Нет подключения" + кешированные данные из localStorage (только просмотр)
Медленный 3GSkeleton screens при загрузке. Кнопки disabled пока данные не загружены. Таймаут 10 сек → ошибка с кнопкой повтора

07 — Animation & Motion
Анимации и движение
Принципы motion design для Telegram Mini App. Минимальны, но заметны. Функциональны, не декоративны.
⚡
Микро-взаимодействия
Тап на слот → немедленное выделение (синий фон). Тап на день → плавная смена цвета. Ощущение отзывчивости без задержки.
transition: background 150ms ease, transform 100ms ease;
active: scale(0.96)
🔄
Переходы между экранами
Slide-in справа при переходе вперёд. Slide-out влево при "Назад". Нет прыжков, нет мигания — плавное скольжение.
transform: translateX(100%) → translateX(0);
duration: 250ms · ease-out
✨
Success анимация
Иконка ✅ появляется с scale(0.5) → scale(1.1) → scale(1). После этого — stagger reveal деталей встречи сверху вниз.
keyframes: scale(0.5) → scale(1.1) → scale(1);
duration: 400ms · spring easing
stagger delay cards: 80ms
💀
Skeleton Loading
Серые плейсхолдеры кнопок и карточек. Shimmer-анимация (opacity 0.5 → 1 → 0.5). Появляются мгновенно, исчезают fade-out при загрузке данных.
@keyframes shimmer { 0%,100%{opacity:0.4} 50%{opacity:0.8} }
animation: shimmer 1.5s ease-in-out infinite
📅
Появление слотов
При выборе дня — слоты появляются с stagger анимацией. Каждый следующий слот появляется с задержкой 30ms. Создаёт ощущение "загрузки данных".
animation: fadeInUp 200ms ease-out;
animation-delay: calc(var(--i) * 30ms);
@keyframes fadeInUp { from { opacity:0; transform: translateY(8px) } }
🎯
Haptic Feedback
Tactile response через Telegram SDK. При выборе слота — medium impact. При успешном бронировании — success notification. Не добавлять визуальный шум поверх.
tg.HapticFeedback.impactOccurred('medium') // слот
tg.HapticFeedback.notificationOccurred('success') // бронь
tg.HapticFeedback.notificationOccurred('error') // ошибка
Запрещённые анимации
ЗапрещеноПричинаАльтернатива
Parallax эффектыТелефон и так в руках — дополнительное движение вызывает тошнотуСтатичные элементы, только opacity transitions
duration > 400msОщущение лагов. Пользователь думает, что нажал не тудаВсе transitions ≤ 300ms
Бесконечные декоративные анимацииПерегружают процессор Telegram WebView, садят батареюАнимации только при user actions
Bounce-анимацииВыглядят несерьёзно для профессионального инструментаease-out · ease-in-out · cubic-bezier для упругости только в success state
