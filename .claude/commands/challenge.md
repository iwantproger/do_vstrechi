# /challenge — Челлендж кода

Проверь текущий код на прочность:

1. Прочитай ключевые файлы проекта:
   - `backend/` — роуты, модели, авторизация
   - `bot/` — хэндлеры, FSM, планировщик напоминаний
   - `frontend/index.html` — клиентская логика, XSS-защита
   - `nginx/nginx.conf` — rate limiting, заголовки безопасности
   - `docker-compose.yml` — сеть, volumes, переменные окружения

2. Найди 3 самых слабых места в проекте:
   - Для каждого объясни:
     - Что может пойти не так
     - Как это воспроизвести (конкретный сценарий)
     - Как это исправить (конкретный код или конфигурация)

3. Области для поиска:
   - Race conditions в asyncpg (параллельные запросы на один слот)
   - Unhandled exceptions в reminder_loop (бот упадёт тихо)
   - Telegram API rate limits (flood wait)
   - Потеря данных при перезапуске контейнеров
   - Отсутствие валидации входных данных на граничных значениях
   - Незащищённые эндпоинты (публичные без авторизации)
   - Connection pool exhaustion при нагрузке (min_size=2, max_size=10)

4. Предложи одно архитектурное улучшение, которое сделает проект надёжнее

Будь жёстким критиком — лучше найти проблемы сейчас, чем потом в проде.
