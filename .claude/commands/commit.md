# /commit — Умный коммит

Выполни следующие шаги:

1. Запусти `git diff --staged` и `git diff` чтобы увидеть все изменения
2. Определи тип изменения:
   - `feat:` — новая функциональность
   - `fix:` — исправление бага
   - `refactor:` — рефакторинг без изменения поведения
   - `style:` — изменения стилей, форматирования
   - `docs:` — документация
   - `chore:` — рутинные задачи, настройки
   - `security:` — исправления безопасности
3. Напиши сообщение коммита в формате: `тип: краткое описание на русском`
4. Добавь все изменённые отслеживаемые файлы в staging (`git add -u`)
5. Создай коммит
6. Покажи результат `git log --oneline -3`

Не спрашивай подтверждения — просто делай.
