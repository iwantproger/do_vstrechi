# /review — Ревью кода перед коммитом

1. Запусти `git diff` и `git diff --staged` — изучи все изменения

2. Для каждого изменённого файла проверь:
   - Нет ли хардкода токенов, паролей, IP-адресов, секретных ключей
   - Нет ли синхронных I/O операций (`requests`, `psycopg2`, `time.sleep`)
   - Соблюдён ли `async/await` везде где нужно
   - Нет ли прямых запросов из бота в PostgreSQL (только через Backend API)
   - Нет ли `allow_origins=["*"]` в CORS
   - Нет ли `telegram_id` из query params/body для авторизации (должен быть из `Depends`)
   - Нет ли `innerHTML` с непроверенными данными без `escHtml()`
   - Нет ли SQL с f-строками (только параметры `$1, $2`)
   - Нет ли TODO/FIXME которые блокируют работу фичи

3. Проверь соответствие 152-ФЗ:
   - Нет ли обращений к зарубежным сервисам для хранения персональных данных пользователей

4. Оцени готовность:
   - ✅ Готово к коммиту
   - ⚠️ Есть замечания (некритичные, можно коммитить)
   - ❌ Нельзя коммитить (критические проблемы)

5. Если есть проблемы — покажи конкретные `файл:строка` и как исправить

Будь строгим — лучше найти проблему сейчас, чем в продакшене.
